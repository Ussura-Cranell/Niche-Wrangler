# Niche Wrangler

Инструмент для анализа ниш на маркетплейсах на основе выгрузок таблиц. Проект автоматизирует сбор, очистку, фильтрацию и статистический анализ данных о товарах, продавцах и брендах.

## Описание

Скрипт предназначен для маркетологов и аналитиков, работающих с большими массивами данных товарных карточек. Он позволяет быстро оценить привлекательность ниши, выявить ключевых игроков, проанализировать ценовые сегменты и популярные поисковые запросы (n-граммы).

Ключевые особенности:
- **Автоматическое кэширование**: повторный запуск без изменений в исходных данных происходит мгновенно.
- **Гибкая фильтрация**: поддержка сложных логических правил (AND, OR, NOT) для выделения конкретных подниш.
- **Бизнес-аналитика**: расчет доли рынка, индекса Херфиндаля — Хиршмана (HHI), распределения по ценам.
- **Лингвистический анализ**: частотный анализ слов и фраз в названиях товаров.

## Быстрый старт

### Шаг 1. Подготовка данных
Поместите файлы таблиц (CSV) в папку `data/`. 
Скрипт автоматически определит и переименует колонки (Название, Артикул, Выручка, Продажи, Цена, Бренд, Продавец).

### Шаг 2. Первичный анализ (без фильтров)
Запустите скрипт в автоматическом режиме:
```bash
python main.py
```
Скрипт выполнит:
1. Архивацию исходных файлов в `archive/`.
2. Слияние всех CSV в один файл в `merged_data/`.
3. Лингвистический анализ (топ слов и фраз) по всему массиву данных.
4. Сохранение результатов в `ngram_analysis/`.

*При повторном запуске без изменений в папке `data/` шаги архивации и слияния будут пропущены.*

### Шаг 3. Настройка фильтрации
Если вас интересует конкретная ниша, создайте правила фильтрации.

**Вариант А: Ручное редактирование**
Отредактируйте файл `rules.yaml`. Пример структуры:
```yaml
version: '2.0'
mode: multi
subsets:
  my_niche:
    include:
      operator: AND
      conditions:
        - type: contains
          value: полка
        - type: contains
          value: угловая
    exclude:
      operator: OR
      conditions:
        - type: contains
          value: пластиковая
```

**Вариант Б: Интерактивный конструктор**
Запустите конструктор правил:
```bash
python src/rule_builder.py
```
Следуйте инструкциям в терминале для создания дерева условий. Результат будет сохранен в `rules.yaml`.

### Шаг 4. Анализ отфильтрованной ниши
После создания `rules.yaml` снова запустите:
```bash
python main.py
```
Скрипт обнаружит файл правил, применит фильтр к данным и пересчитает n-граммы только для отобранной выборки. Результаты сохранятся в `filtered_subsets/` и `ngram_analysis/`.

Вы можете менять `rules.yaml` и перезапускать `python main.py` столько раз, сколько нужно. Тяжелые операции (слияние) выполняться не будут.

### Шаг 5. Финальный отчет
Когда фильтрация завершена и результаты вас устраивают, сформируйте итоговый бизнес-отчет:
```bash
python main.py --stats
```
Команда выполнит:
1. Загрузку последних отфильтрованных данных.
2. Расчет общей выручки, количества продаж, числа уникальных продавцов и брендов.
3. Определение топ-10 продавцов и брендов с долей рынка >1%.
4. Расчет индекса концентрации HHI.
5. Экспорт ценовых интервалов (квантилей) в `reports/price_intervals.csv`.
6. Архивацию текущей версии `rules.yaml` в папку `archive/` (для истории изменений).

### Очистка рабочего пространства
Для удаления всех временных файлов, кэша и результатов анализа (кроме архива исходных данных) используйте:
```bash
python main.py --clear
```
Папка `data/` будет очищена, что позволит начать работу с новым набором файлов.

## Подробное описание модулей

### Оркестратор (main.py)
Управляет потоком выполнения. Проверяет наличие артефактов (CSV-файлов в целевых папках) перед запуском каждого шага. Если данные не изменились (проверка по MD5-хешу метаинформации файлов в `data/`), шаг пропускается, и данные загружаются из кэша или предыдущих этапов.

### Нормализация данных (column_normalizer.py, data_loader.py)
- **column_normalizer**: Содержит словарь маппинга (`COLUMN_MAPPING`) для приведения разноименных колонок из разных отчетов к единому стандарту (`Name`, `SKU`, `Seller`, `Brand`, `Revenue`, `Sales`, `Price`, `Country`).
- **data_loader**: Читает все CSV из `data/`, применяет нормализацию, очищает числовые поля от лишних символов (пробелы, валюты), удаляет дубликаты по `SKU` и сохраняет результат.

### Фильтрация (rule_engine.py, rule_builder.py)
- **rule_engine**: Парсит `rules.yaml` и компилирует условия в объекты-маски Pandas. Поддерживает вложенные логические операторы. Режим `multi` позволяет выделять несколько независимых ниш, режим `first` — последовательное исключение.
- **rule_builder**: CLI-утилита для пошагового создания сложных правил без ручного написания YAML.

### Аналитика (ngram_analyzer.py, niche_analyzer.py)
- **ngram_analyzer**: Токенизирует названия товаров, удаляет стоп-слова, применяет упрощенный стеммер и считает частоту встречаемости униграмм, биграмм и триграмм.
- **niche_analyzer**: Агрегирует данные по продавцам и брендам, считает доли выручки, экспортирует распределение цен по интервалам (равное количество товаров или равная ширина интервала).

## Формат правил (rules.yaml)

Файл состоит из трех основных секций:
1. `version`: версия формата (текущая '2.0').
2. `mode`: режим применения (`multi` или `first`).
3. `subsets`: словарь именованных выборок.

Каждая выборка может содержать:
- `include`: условия включения (обязательно).
- `exclude`: условия исключения (опционально).

Типы условий:
- `contains`: проверка наличия подстроки.
- `regex`: проверка по регулярному выражению.
- `operator`: логическая группировка (`AND`, `OR`, `NOT`).

## Устранение неполадок

- **Ошибка "Колонка 'Country' отсутствует"**: В некоторых выгрузках нет колонки страны. Это предупреждение не критично, анализ продолжится без учета стран.
- **Пустой результат после фильтрации**: Проверьте `rules.yaml`. Убедитесь, что условия `include` не слишком строгие или не содержат опечаток. Используйте `python main.py` без правил сначала, чтобы увидеть общие n-граммы и понять, какие слова реально встречаются в данных.
- **Ошибки кодировки**: Скрипт ожидает файлы в кодировке `utf-8-sig` (стандарт для Excel/CSV из РФ). Если ваши файлы в другой кодировке, измените параметр `encoding` в `data_loader.py`.
